# Flower Dance 演示模式指南

## 概述

本指南提供 Flower Dance 的标准演示流程，确保在产品演示、项目汇报和答辩过程中能够稳定输出高质量结果。

---

## 演示环境准备

### 启动方式

```bash
cd backend
python main.py
```

前端访问：`http://localhost:8000`

### 进入演示模式

1. 在首页点击「进入演示模式」按钮
2. 选择一个演示用户（或创建空白演示）
3. 系统自动加载预设数据

---

## 标准演示流程（5分钟版）

### 步骤 1：上传文本（30秒）

- 点击左侧「种花」面板
- 在文本框中输入内容，或点击示例文本快速填充
- 点击「种下这段记忆」按钮
- 预期效果：系统自动解析文本并生成认知卡片

### 步骤 2：查看十二花神分类（45秒）

- 中间花海区域展示生成的花朵
- 点击任意花朵查看对应认知卡片详情
- 观察不同颜色代表的花神维度
- 预期效果：看到完整的十二花神视觉分布

### 步骤 3：查看全局画像（45秒）

- 点击右侧「拾花」面板
- 查看全局画像图
- 观察各维度节点大小和连接线
- 预期效果：看到完整的个人认知网络图谱

### 步骤 4：提问对话（1分钟）

- 在聊天输入框输入问题
- 推荐问题："最近我最大的变化是什么？"
- 点击发送按钮
- 预期效果：AI 基于认知卡片生成个性化回答，相关节点高亮显示

### 步骤 5：查看成长总结（1分钟）

- 提问："我的成长方向是什么？"
- 查看回答内容
- 预期效果：获得基于数据的成长建议

### 步骤 6：切换演示用户（30秒）

- 点击右上角「退出」按钮
- 重新进入演示模式，选择另一个用户
- 预期效果：快速切换到不同用户的完整数据

## 离线演示注意事项

1. **数据文件位置**：`demo_data/users/` 目录下存放所有演示用户数据
2. **无需网络**：演示数据已预先生成，可离线运行
3. **恢复数据**：退出演示模式后自动恢复原始数据
4. **演示文档**：`demo_data/docs/` 目录下有可上传的演示文档

## 演示技巧

1. **时间控制**：每个步骤严格控制时间，总时长不超过 5 分钟
2. **重点突出**：选择最能展示核心功能的环节重点演示
3. **互动引导**：邀请观众提问，展示对话功能
4. **数据对比**：切换不同用户展示系统的普适性
5. **故事讲述**：结合用户画像讲述生动的个人故事

## 故障排查

### 演示用户加载失败
- 检查 `demo_data/users/` 目录是否存在
- 确认文件格式为有效的 JSON

### 认知卡片不显示
- 检查后端服务是否正常运行
- 确认 API 响应数据格式正确

### 退出演示模式后数据异常
- 系统会自动备份和恢复数据
- 如出现问题，可重启后端服务
